# A2A 智能体互联

> ⚠️ **重要**：A2A 是此前设计完全缺失的子系统，用于**跨进程、跨服务**的 Agent 通信，与 SubAgent（进程内嵌套委托）互补。

---

## 1. 架构定位

```
┌─────────────────────────────────────────────────────────────────┐
│                    A2A vs SubAgent 边界                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  SubAgent（进程内委托）                                          │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ • 父子 Agent 在同一进程                                   │   │
│  │ • 通过 Observable<AgentEvent> 通信                        │   │
│  │ • 共享内存、共享销毁信号                                   │   │
│  │ • 同步调用、低延迟                                        │   │
│  │ • 适用：任务分解、专家委托                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  A2A（跨进程通信）                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ • Agent 可能在不同进程/机器                                │   │
│  │ • 通过 Transport 传输消息                                  │   │
│  │ • 独立生命周期、独立销毁                                   │   │
│  │ • 异步调用、需处理网络问题                                 │   │
│  │ • 适用：分布式 Agent 系统、多服务协作                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 消息信封标准化

所有 A2A 通信使用统一的信封格式：

```typescript
// src/a2A/types.ts

/** A2A 消息信封（Zod Schema） */
export const A2AMessageSchema = z.object({
  // === 路由信息 ===
  /** 消息唯一 ID（用于幂等去重） */
  id: z.string().uuid(),
  
  /** 发送方 Agent ID */
  from: z.string(),
  
  /** 接收方 Agent ID（'*' = 广播） */
  to: z.string(),
  
  /** 时间戳（毫秒） */
  timestamp: z.number().int().positive(),
  
  // === 可靠性 ===
  /** 消息过期时间（毫秒，0 = 永不过期） */
  ttl: z.number().int().nonnegative().default(0),
  
  /** 关联 ID（用于请求-响应模式） */
  correlationId: z.string().optional(),
  
  /** 序列号（用于有序消息） */
  sequence: z.number().int().nonnegative().optional(),
  
  // === 安全 ===
  /** 消息签名（可选） */
  signature: z.string().optional(),
  
  /** 协议版本 */
  version: z.string().default('1.0.0'),
  
  // === 内容 ===
  /** 消息类型 */
  type: z.enum([
    'request',      // 请求
    'response',     // 响应
    'notification', // 单向通知
    'error',        // 错误
    'heartbeat',    // 心跳
    'ack',          // 确认
  ]),
  
  /** 消息载荷 */
  payload: z.unknown(),
  
  /** 元数据 */
  metadata: z.record(z.string(), z.unknown()).optional(),
});

export type A2AMessage = z.infer<typeof A2AMessageSchema>;

/** A2A 错误信封 */
export const A2AErrorSchema = z.object({
  id: z.string().uuid(),
  from: z.string(),
  to: z.string(),
  timestamp: z.number(),
  correlationId: z.string(),
  type: z.literal('error'),
  payload: z.object({
    code: z.string(),
    message: z.string(),
    details: z.unknown().optional(),
  }),
});

export type A2AError = z.infer<typeof A2AErrorSchema>;

/** A2A 确认信封 */
export const A2AAckSchema = z.object({
  id: z.string().uuid(),
  from: z.string(),
  to: z.string(),
  timestamp: z.number(),
  correlationId: z.string(),
  type: z.literal('ack'),
  payload: z.object({
    originalMessageId: z.string(),
    status: z.enum(['received', 'processing', 'completed', 'failed']),
  }),
});

export type A2AAck = z.infer<typeof A2AAckSchema>;
```

## 3. 传输层抽象

```typescript
// src/a2a/transport.ts

/** 传输层连接状态 */
export type TransportStatus = 
  | 'disconnected'
  | 'connecting'
  | 'connected'
  | 'reconnecting'
  | 'error';

/** 传输层配置 */
export interface TransportConfig {
  /** 连接超时（毫秒） */
  connectTimeout?: number;
  
  /** 重连配置 */
  reconnect?: {
    enabled: boolean;
    maxAttempts: number;
    initialDelay: number;
    maxDelay: number;
    backoffMultiplier: number;
  };
  
  /** 心跳配置 */
  heartbeat?: {
    enabled: boolean;
    interval: number;
    timeout: number;
  };
  
  /** 消息积压配置 */
  backlog?: {
    maxSize: number;
    overflowStrategy: 'drop' | 'block' | 'replace';
  };
}

/** 传输层接口 */
export interface A2ATransport {
  /** 传输层名称 */
  readonly name: string;
  
  /** 连接状态流 */
  readonly status$: Observable<TransportStatus>;
  
  /** 当前状态 */
  readonly status: TransportStatus;
  
  /** 连接 */
  connect(): Promise<void>;
  
  /** 断开 */
  disconnect(): Promise<void>;
  
  /** 发送消息 */
  send(message: A2AMessage): Promise<void>;
  
  /** 接收消息流 */
  readonly messages$: Observable<A2AMessage>;
  
  /** 销毁 */
  destroy(): void;
}

/** 传输层工厂 */
export interface TransportFactory {
  create(config: TransportConfig): A2ATransport;
}
```

## 4. A2A 客户端

> ⚠️ **P3 设计说明**：A2A 是**对等网络（Peer-to-Peer）**模式，而非传统客户端-服务器模式。

```
┌─────────────────────────────────────────────────────────────────┐
│                    A2A P2P 通信模式                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Agent A (A2AClient)                    Agent B (A2AClient)      │
│  ┌─────────────────────┐               ┌─────────────────────┐   │
│  │ request(to='B')     │ ───────────→  │ messages$ (receive) │   │
│  │                     │               │                     │   │
│  │ response$ (receive) │ ←───────────  │ respond(request)    │   │
│  └─────────────────────┘               └─────────────────────┘   │
│                                                                  │
│  双方都是对等端点：                                               │
│  • 都可以发起 request / notify / broadcast                       │
│  • 都可以响应收到的 request（通过 respond）                        │
│  • 都可以订阅 messages$ 接收消息                                  │
│                                                                  │
│  A2AServer 是传输层：                                            │
│  • 管理 WebSocket 连接                                           │
│  • 路由消息到正确的 Agent                                         │
│  • 不参与业务逻辑                                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

/** 扩展 AgentContext */
export interface AgentContext {
  // ... 现有字段
  
  /** A2A 客户端（可选） */
  a2a?: A2AClient;
}

/** A2A 工具：向其他 Agent 发送请求 */
export function createA2ARequestTool(client: A2AClient): Tool {
  return {
    name: 'a2a_request',
    description: 'Send a request to another agent and wait for response',
    parameters: z.object({
      agent_id: z.string().describe('Target agent ID'),
      payload: z.unknown().describe('Request payload'),
      timeout: z.number().optional().describe('Timeout in ms'),
    }),
    execute: async (args, ctx: ToolContext): Promise<ToolResult> => {
      const response = await client.request(args.agent_id, args.payload, {
        timeout: args.timeout,
      });
      
      return {
        type: 'a2a_response',
        content: response.payload,
        metadata: {
          from: response.from,
          correlationId: response.correlationId,
        },
      };
    },
  };
}

## 7. 背压与流量控制

```typescript
// src/a2a/operators.ts

/** 背压控制操作符 */
export function a2aBackpressure<T>(
  options: {
    maxSize: number;
    overflowStrategy: 'drop' | 'block' | 'sample';
    sampleInterval?: number;
  },
): OperatorFunction<T, T> {
  let buffer: T[] = [];
  
  return pipe(
    tap({
      next: (value) => {
        if (buffer.length >= options.maxSize) {
          switch (options.overflowStrategy) {
            case 'drop':
              return; // 丢弃
            case 'block':
              throw new Error('Buffer full');
            case 'sample':
              buffer.shift(); // 移除最旧的
              break;
          }
        }
        buffer.push(value);
      },
    }),
    // 使用 bufferTime 或 bufferCount 控制发射速率
    bufferTime(100), // 每 100ms 发射一批
    mergeAll(), // 展平
  );
}

/** 断线重连操作符 */
export function a2aReconnect<T>(
  transport: A2ATransport,
  options: {
    maxAttempts: number;
    initialDelay: number;
    maxDelay: number;
  },
): OperatorFunction<T, T> {
  return (source) => {
    return source.pipe(
      retry({
        count: options.maxAttempts,
        delay: (error, attempt) => {
          const delay = Math.min(
            options.initialDelay * attempt,
            options.maxDelay,
          );
          return timer(delay);
        },
        resetOnSuccess: true,
      }),
    );
  };
}

/** 幂等去重操作符 */
export function a2aDeduplicate<T extends { id: string }>(
  ttl: number = 300000,
): OperatorFunction<T, T> {
  const seen = new Map<string, number>();
  
  return pipe(
    filter((message) => {
      const now = Date.now();
      
      // 清理过期记录
      for (const [id, timestamp] of seen) {
        if (now - timestamp > ttl) {
          seen.delete(id);
        }
      }
      
      if (seen.has(message.id)) {
        return false; // 重复，过滤掉
      }
      
      seen.set(message.id, now);
      return true;
    }),
  );
}
```

## 8. A2A 与 MsgHub 的关系

```
┌─────────────────────────────────────────────────────────────────┐
│                    Agent 通信方式对比                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  MsgHub（进程内广播）                                            │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ • 同一进程内的 Agent 间通信                              │   │
│  │ • 共享 Subject，零网络开销                               │   │
│  │ • 无消息格式约束                                         │   │
│  │ • 无可靠性机制                                           │   │
│  │ • 适用：工作流编排、团队协作                              │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  A2A（跨进程通信）                                               │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ • 不同进程/机器的 Agent 间通信                           │   │
│  │ • 通过 Transport，需网络                                 │   │
│  │ • 严格消息信封格式                                       │   │
│  │ • 完整可靠性机制（重连、背压、幂等）                       │   │
│  │ • 适用：分布式系统、微服务 Agent                          │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  选择原则：                                                       │
│  • Agent 在同一进程 → MsgHub                                    │
│  • Agent 在不同进程 → A2A                                       │
│  • 混合场景 → MsgHub 用于本地协作，A2A 用于远程调用              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 9. 设计约束

| 约束 | 描述 | 违反后果 |
|------|------|---------|
| **消息格式统一** | 所有 A2A 消息必须使用 `A2AMessage` 信封 | 解析失败、路由混乱 |
| **幂等处理** | 接收方必须去重（基于 `message.id`） | 消息重复处理 |
| **超时必设** | 请求必须有 `ttl` 或显式超时 | 请求永久挂起 |
| **签名可选** | 跨网络消息应签名验证 | 消息伪造、中间人攻击 |
| **传输抽象** | 业务代码只依赖 `A2ATransport` 接口 | 换传输层需改代码 |
| **背压必设** | 发送方必须配置 `backlog` 策略 | 消息积压 OOM |
| **重连必设** | 长连接必须配置 `reconnect` 策略 | 断线后无法恢复 |
