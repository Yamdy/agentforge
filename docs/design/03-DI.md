# 轻量依赖注入

> 本文档定义 AgentForge 的轻量 DI 设计：依赖倒置、三层 Context 结构、ContextBuilder。

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **依赖倒置** | 核心 Loop / 钩子系统只依赖 Interface，不依赖具体实现 |
| **构造函数注入** | 所有外部能力通过构造函数注入，禁止内部 `new` 硬编码 |
| **不引入 IoC 容器** | 无装饰器、无反射、无容器类（Inversify/TypeDI 一律不用） |
| **Context 闭包传递** | 依赖通过闭包传入事件循环处理器，不作为事件载荷 |
| **配置驱动装配** | `createAgent(config)` 自动解析依赖 |
| **Builder 兜底** | 编程式用户用 ContextBuilder 手动组装 |
| **测试友好** | Mock 接口替换即可 |

---

## 2. 依赖倒置：核心只依赖接口

核心 Loop 和所有事件处理器只依赖抽象接口，不依赖具体实现。LLM、Memory、ToolRegistry、MCPRouter、SubagentRegistry 全部抽象：

```typescript
// ✅ 正确：Loop 只依赖接口
class Agent {
  constructor(
    private config: AgentConfig,
    private ctx: AgentContext,   // 全部通过 Context 注入
  ) {}
}

// ❌ 错误：硬编码具体实现
class Agent {
  private llm = new OpenAIAdapter();  // 禁止！
}
```

---

## 3. 依赖作用域划分（关键）

> 作用域混乱 = 多 Agent 上下文串数据、内存暴涨

```
┌─────────────────────────────────────────────────────────────────┐
│                  ApplicationServices（全局单例）                  │
│  应用启动时创建一次，所有 Agent 共享                               │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────────┐  │
│  │ Tracer      │ │ Metrics     │ │ SchemaRegistry          │  │
│  │ (全局打点)   │ │ (全局计数)   │ │ (事件 Schema 注册)        │  │
│  └─────────────┘ └─────────────┘ └─────────────────────────┘  │
│  ┌─────────────┐ ┌─────────────────────────────────────────┐  │
│  │ LLMFactory  │ │ ToolRegistry (全局注册表)                  │  │
│  │ (工厂方法)   │ │                                          │  │
│  └─────────────┘ └─────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                     AgentContext（会话级实例）                    │
│  每次 createAgent 创建，单个 Agent 独立                           │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────────┐  │
│  │ sessionId   │ │ memory      │ │ pauseController         │  │
│  │ agentName   │ │ (会话记忆)   │ │ (暂停/恢复控制器)          │  │
│  └─────────────┘ └─────────────┘ └─────────────────────────┘  │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ llm (通过 Factory 创建)     tools (引用全局注册表)          │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      ToolContext（瞬时，用完即弃）                │
│  单次 Tool 执行时创建，不缓存                                     │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ toolCallId │ parentSessionId │ timeout │ signal          │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 作用域风险

| 错误做法 | 后果 |
|---------|------|
| 会话级状态放到全局 | 多 Agent 共享 memory → 上下文串数据 |
| 全局服务每次 `new` | 资源浪费、连接池失效 |
| 瞬时上下文复用 | 取消信号混乱、timeout 错位 |

export interface LLMAdapter {
  chat(messages: Message[], options?: LLMOptions): Promise<LLMResponse>;
  stream(messages: Message[], options?: LLMOptions): AsyncGenerator<LLMChunk>;
}

export interface LLMAdapterFactory {
  create(config: ModelConfig): LLMAdapter;

export interface ToolRegistry {

  execute(name: string, args: Record<string, unknown>, ctx?: ToolContext): Promise<string>;

}

export interface MemoryStore {
  add(message: Message): void;
  getAll(): Message[];
  getRecent(count: number): Message[];
  clear(): void;

// ========== 检查点 ==========

export interface CheckpointStorage {
  save(checkpoint: Checkpoint): Promise<void>;
  load(sessionId: string): Promise<Checkpoint | null>;
  list(sessionId?: string): Promise<Checkpoint[]>;
  delete(id: string): Promise<void>;
}

// ========== 可观测性 ==========

export interface Tracer {
  startSpan(name: string, options?: { attributes?: Record<string, unknown>; parent?: string }): string;
  endSpan(spanId: string, options?: { code?: string }): void;
  addEvent(spanId: string, name: string, attributes?: Record<string, unknown>): void;
  recordException(spanId: string, error: Error): void;
}

export interface Metrics {
  increment(name: string, value?: number, tags?: Record<string, string>): void;
  histogram(name: string, value: number, tags?: Record<string, string>): void;
}

// ========== HITL ==========

export interface HITLAskOptions {
  question: string;
  askId: string;
  toolCallId: string;
  toolName: string;
  options?: string[];
  metadata?: Record<string, unknown>;
}

export interface HITLController {
  /** Ask a question - returns Promise that resolves when human responds */
  ask(options: HITLAskOptions): Promise<string>;
  /** Subscribe to asks (for UI to observe) */
  onAsk(handler: (ask: { askId: string; question: string; options?: string[] }) => void): () => void;
  /** Provide an answer (for UI to call) */
  answer(askId: string, answer: string): void;
}

export interface PauseController {
  pause(): void;
  resume(): void;
  isPaused(): boolean;
  onResume(): Promise<void>;
}

export interface MCPClient {
  connect(): Promise<void>;
  disconnect(): Promise<void>;
  tools(): Promise<Tool[]>;
  callTool(name: string, args: Record<string, unknown>): Promise<string>;
  status(): 'connected' | 'disconnected' | 'error';
}

// ========== Subagent ==========

export interface SubagentRegistry {
  has(name: string): boolean;
  run(name: string, input: string, options?: { sessionMessages?: Message[] }): Promise<string>;
  list(): Array<{ name: string; mode: AgentMode; description?: string }>;
}

// ========== 瞬时上下文 ==========

export interface ToolContext {
  toolCallId: string;
  parentSessionId: string;
  timeout?: number;
  signal?: AbortSignal;

## 5. 三层 Context 结构

```typescript
// src/core/context.ts

// ========== 全局单例（应用级，跨 Agent 共享） ==========

export interface ApplicationServices {
  /** 全局打点 */
  tracer?: Tracer;
  /** 全局指标 */
  metrics?: Metrics;
  /** 事件 Schema 注册中心 */
  schemaRegistry: SchemaRegistry;
  /** LLM 适配器工厂（按配置创建实例） */
  llmFactory: LLMAdapterFactory;
  /** LLM Provider 注册表（全局共享） */
  providerRegistry: ProviderRegistry;
  /** 全局工具注册表（共享） */
  toolRegistry: ToolRegistry;
}

// ========== 会话级实例（单 Agent 独立） ==========

export interface AgentContext {
  // --- 身份 ---
  sessionId: string;
  agentName: string;

  // --- 会话独立状态（每 Agent 新建实例） ---
  memory: MemoryStore;
  pauseController: PauseController;

  // --- 引用全局服务（不新建） ---
  services: ApplicationServices;

  // --- 适配器实例（可能共享，可能独立） ---
  llm: LLMAdapter;                    // 通过 Factory 创建
  tools: ToolRegistry;                 // 通常引用全局注册表

  // --- 可选依赖（按配置注入） ---
  checkpoint?: CheckpointStorage;
  hitl?: HITLController;
  mcp?: MCPClient;
  subagents?: SubagentRegistry;

  // --- 安全（可选 — 未配置则零开销） ---
  permissionPolicy?: PermissionPolicy;
  permissionController?: PermissionController;
  sandboxExecutor?: SandboxExecutor;
  auditLogger?: AuditLogger;
  rateLimiter?: RateLimiter;
  inputSanitizer?: InputSanitizer;

  // --- 内存管理（可选） ---
  compactionManager?: CompactionManager;

  // --- 控制信号 ---
  abortSignal?: AbortSignal;

  // --- 错误钩子 ---
  onError?: (error: Error, event: AgentEvent, category: ErrorCategory) => void;
}

// ========== 瞬时上下文（单次调用，用完即弃） ==========

// ToolContext 已在 interfaces.ts 中定义，此处不再重复
```

## 6. ContextBuilder

```typescript
// src/core/context-builder.ts

export class ContextBuilder {
  private context: Partial<AgentContext> = {};
  private appServices?: ApplicationServices;

  static create(): ContextBuilder {
    return new ContextBuilder();
  }

  /** 注入全局服务（必须最先设置，或由 createAgent 自动注入） */
  withAppServices(services: ApplicationServices): this {
    this.appServices = services;
    this.context.services = services;
    return this;
  }

  withSessionId(sessionId: string): this {
    this.context.sessionId = sessionId;
    return this;
  }

  withAgentName(name: string): this {
    this.context.agentName = name;
    return this;
  }

  withLLM(llm: LLMAdapter): this {
    this.context.llm = llm;
    return this;
  }

  withTools(tools: ToolRegistry | Tool[]): this {
    this.context.tools = Array.isArray(tools)
      ? new SimpleToolRegistry(tools)
      : tools;
    return this;
  }

  withMemory(memory: MemoryStore): this {
    this.context.memory = memory;
    return this;
  }

  withPauseController(controller: PauseController): this {
    this.context.pauseController = controller;
    return this;
  }

  withCheckpoint(storage: CheckpointStorage): this {
    this.context.checkpoint = storage;
    return this;
  }

  withHITL(hitl: HITLController): this {
    this.context.hitl = hitl;
    return this;
  }

  withMCP(mcp: MCPClient): this {
    this.context.mcp = mcp;
    return this;
  }

  withSubagents(subagents: SubagentRegistry): this {
    this.context.subagents = subagents;
    return this;
  }

  withAbortSignal(signal: AbortSignal): this {
    this.context.abortSignal = signal;
    return this;
  }

  withErrorHandler(handler: (error: Error, event: AgentEvent, category: ErrorCategory) => void): this {
    this.context.onError = handler;
    return this;
  }

  withQuota(quota: QuotaController): this {
    this.context.services.quota = quota;
    return this;
  }

  withCompactionManager(manager: CompactionManager): this {
    this.context.compactionManager = manager;
    return this;
  }

  build(): AgentContext {
    if (!this.context.llm) throw new Error('LLM adapter is required');
    if (!this.context.tools) throw new Error('ToolRegistry is required');

    return {
      sessionId: this.context.sessionId ?? generateId(),
      agentName: this.context.agentName ?? 'agent',
      memory: this.context.memory ?? new InMemoryStore(),
      pauseController: this.context.pauseController ?? new DefaultPauseController(),
      services: this.context.services ?? createDefaultAppServices(),
      llm: this.context.llm,
      tools: this.context.tools,
      checkpoint: this.context.checkpoint,
      hitl: this.context.hitl,
      mcp: this.context.mcp,
      subagents: this.context.subagents,
      abortSignal: this.context.abortSignal,
      onError: this.context.onError,
    };
  }
}
```

## 7. 生命周期管理

```typescript
// src/core/lifecycle.ts

// ========== 应用级：启动时创建一次 ==========

export function createApplicationServices(
  config?: Partial<AppConfig>
): ApplicationServices {
  return {
    tracer: config?.tracing ? createTracer(config.tracing) : undefined,
    metrics: config?.metrics ? createMetrics(config.metrics) : undefined,
    schemaRegistry: new SchemaRegistry(),
    llmFactory: createLLMAdapterFactory(config?.models),
    toolRegistry: new SimpleToolRegistry(),
  };
}

// ========== 会话级：每次 createAgent 创建 ==========

export function createAgent(
  config: AgentConfig,
  app?: ApplicationServices,
): Agent {
  const parsed = AgentConfigSchema.parse(config);

  // 获取或创建全局服务
  const services = app ?? createApplicationServices();

  // 会话独立实例
  const memory = new InMemoryStore();
  const pauseController = new DefaultPauseController();

  // LLM 通过工厂创建（可能共享连接池，但是独立实例）
  const llm = services.llmFactory.create(parsed.model);

  // 工具注册表：引用全局 + 追加配置级工具
  const tools = new DelegatingToolRegistry(services.toolRegistry);
  parsed.tools.forEach((t) => {
    if (typeof t === 'string') tools.register(resolveBuiltinTool(t));
    else tools.register(t);
  });

  // 可选依赖
  const checkpoint = parsed.checkpoint
    ? createCheckpointStorage(
        typeof parsed.checkpoint === 'boolean' ? { storage: 'memory' } : parsed.checkpoint
      )
    : undefined;

  const hitl = parsed.hitl ? createHITLController(parsed.hitl) : undefined;
  const mcp = parsed.mcp ? createMCPClient(parsed.mcp) : undefined;
  const subagents = parsed.subagents ? createSubagentRegistry(parsed.subagents) : undefined;

  // 构建 Context
  const context = ContextBuilder.create()
    .withAppServices(services)
    .withAgentName(parsed.name)
    .withLLM(llm)
    .withTools(tools)
    .withMemory(memory)
    .withPauseController(pauseController)
    .withCheckpoint(checkpoint)
    .withHITL(hitl)
    .withMCP(mcp)
    .withSubagents(subagents)
    .build();

  // 构建 Agent
  const agent = new Agent(
    {
      name: parsed.name,
      maxSteps: parsed.maxSteps,
      systemPrompt: parsed.systemPrompt,
    },
    context,
  );

  // 配置驱动操作符
  if (parsed.timeout) agent.use(timeout(parsed.timeout));
  if (parsed.retry) agent.use(retry(parsed.retry));

  return agent;
}

// ========== 瞬时：Tool 执行时创建 ==========

export function createToolContext(
  agentCtx: AgentContext,
  toolCallId: string,
  options?: { timeout?: number },
): ToolContext {
  return {
    toolCallId,
    parentSessionId: agentCtx.sessionId,
    timeout: options?.timeout,
    signal: agentCtx.abortSignal,
  };
}
```

### 典型使用模式

```typescript
// 1. 应用级：启动时创建一次
const app = createApplicationServices({
  tracing: { exporter: 'otel', endpoint: 'http://localhost:4317' },
  metrics: { prefix: 'agent.' },
});

// 2. 会话级：每次创建 Agent
const agent1 = createAgent({ name: 'coder', model: { provider: 'openai', model: 'gpt-4o' } }, app);
const agent2 = createAgent({ name: 'reviewer', model: { provider: 'anthropic', model: 'claude-3.5' } }, app);

// agent1 和 agent2 共享：
//   app.tracer, app.metrics, app.schemaRegistry, app.toolRegistry
// 但各自独立：
//   memory, pauseController, llm (不同模型)

// 3. 瞬时：Tool 执行时自动创建（开发者不需要手动管理）
agent1.run('Fix the bug');
```

## 8. 禁止重型 DI 容器

| 禁止引入 | 理由 |
|---------|------|
| **Inversify** | 运行时反射开销；装饰器语法增加心智负担；与 Agent 生命周期管理冲突 |
| **TypeDI** | 装饰器强制要求 TypeScript emitDecoratorMetadata；容器托管与 Agent 生命周期冲突 |
| **Awilix** | 函数式注入但容器概念过重；Agent 框架追求轻量部署 |

**为什么不兼容？**

```
Agent 生命周期                       IoC 容器生命周期
─────────────────────               ─────────────────────
run() → 开始                       container.resolve() → 创建
AbortController → 销毁               container.dispose() → 销毁
                                   
问题：                          
- Agent 的 AbortController 触发时，         
  容器不知道执行已取消              
- 容器 dispose 时，                
  循环可能还在 await 步骤              
- 两套生命周期管理 = 竞态风险        
```

**替代方案：轻量手动 DI = ContextBuilder 模式**

- 无运行时反射开销
- Context 闭包传递，生命周期由 `AbortController` 统一管理
- 框架打包体积更小、部署更轻量

## 9. DI 约束清单

| 约束 | 描述 | 违反后果 |
|------|------|---------|
| **核心只依赖接口** | Agent Loop 只依赖 `LLMAdapter` 等接口，不 import 具体实现 | 框架僵化，无法替换 LLM/Memory |
| **禁止内部 `new`** | 外部能力一律构造函数注入 | 隐式耦合，测试困难 |
| **全局服务通过工厂获取** | `app.llmFactory.create()` 而非 `new OpenAIAdapter()` | 连接池失效，资源浪费 |
| **会话状态禁止共享** | `memory`、`pauseController` 必须每 Agent 独立 | 多 Agent 上下文串数据 |
| **瞬时上下文用完即弃** | `ToolContext` 不缓存，每次执行新建 | 取消信号混乱、timeout 错位 |
| **Context 闭包传递** | 不放入事件载荷，避免序列化开销 | 事件循环膨胀，性能下降 |
| **禁止重型 DI 容器** | 不引入 Inversify/TypeDI/Awilix | 生命周期冲突，打包体积膨胀 |
