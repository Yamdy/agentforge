# Error Handling

## Description

Error classes and utilities for HTTP API error responses.

## Classes

### AppError

Base error class with code and HTTP status.

```typescript
class AppError extends Error {
  constructor(code: string, message: string, status: number = 500)
  code: string
  status: number
  toJSON(): { error: { code: string, message: string } }
}
```

### NotFoundError

HTTP 404 error.

### BadRequestError

HTTP 400 error.

### UnauthorizedError

HTTP 401 error.

### ValidationError

HTTP 400 error with field-level validation details.

```typescript
class ValidationError extends Error {
  constructor(errors: { field: string, message: string }[])
  toJSON(): { error: { code, message, details } }
}
```

## Utilities

### ErrorCodes

```typescript
const ErrorCodes = {
  BAD_REQUEST: 400,
  UNAUTHORIZED: 401,
  FORBIDDEN: 403,
  NOT_FOUND: 404,
  INTERNAL_ERROR: 500,
}
```

### isAppError(err)

Type guard for AppError.

### toErrorResponse(err)

Convert any error to JSON response format.

## Examples

```typescript
import { AppError, NotFoundError, toErrorResponse } from 'primo-agent';

// Throw typed error
if (!session) {
  throw new NotFoundError('Session not found');
}

// Convert to response
const response = toErrorResponse(err);
```
