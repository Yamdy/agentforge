# Workflow Orchestration

## Description

Powerful workflow orchestration system for defining and executing complex multi-step agent tasks. Supports sequential pipelines, parallel execution, conditional branching, and agent collaboration via message hub.

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│                    Workflow Builder                       │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐            │
│  │  Sequen- │  │ Parallel │  │  Branch  │            │
│  │   tial   │  │          │  │          │            │
│  └──────────┘  └──────────┘  └──────────┘            │
└────────────────────┬────────────────────────────────────┘
                     │
           ┌─────────▼─────────┐
           │   Committed        │
           │   Workflow         │
           │   (Executable)     │
           └─────────┬─────────┘
                     │
           ┌─────────▼─────────┐
           │   MsgHub           │
           │  (Agent Collab)    │
           └───────────────────┘
```

## Signature

```typescript
function createWorkflow<TInput = unknown>(config: { id: string }): Workflow<TInput, TInput>;

interface Workflow<TInput, TOutput> {
  id: string;
  step<TI, TO>(
    stepId: string,
    step: WorkflowStep<TI, TO>,
    options?: StepOptions
  ): Workflow<TInput, TO>;
  then<TI, TO>(
    stepId: string,
    step: WorkflowStep<TI, TO>,
    options?: StepOptions
  ): Workflow<TInput, TO>;
  parallel<TI, TO>(
    stepIds: string[],
    steps: WorkflowStep<TI, TO>[],
    options?: ParallelOptions
  ): Workflow<TInput, TO[]>;
  branch<TI, TO>(
    condition: (ctx: WorkflowContext) => boolean,
    branches: {
      true: { id: string; step: WorkflowStep<TI, TO> };
      false: { id: string; step: WorkflowStep<TI, TO> };
    },
    options?: BranchOptions
  ): Workflow<TInput, TO>;
  commit(): CommittedWorkflow<TInput, TOutput>;
}

interface CommittedWorkflow<TInput, TOutput> {
  id: string;
  run(input: TInput): Promise<TOutput>;
}
```

## Core Functions

### createWorkflow(config)

Create a new workflow builder.

| Parameter | Type   | Required | Description                |
| --------- | ------ | -------- | -------------------------- |
| config.id | string | Yes      | Unique workflow identifier |

**Returns:** `Workflow<TInput, TInput>` - Workflow builder instance

### createStep(id, execute, options)

Create a reusable workflow step.

| Parameter           | Type                                                                   | Required | Description             |
| ------------------- | ---------------------------------------------------------------------- | -------- | ----------------------- |
| id                  | string                                                                 | Yes      | Step identifier         |
| execute             | (input: TInput, context: WorkflowContext) =&gt; Promise&lt;TOutput&gt; | Yes      | Step execution function |
| options.description | string                                                                 | No       | Step description        |

**Returns:** `WorkflowStep<TInput, TOutput>` - Workflow step

### createAgentStep(id, agent, options)

Create a workflow step that wraps an Agent.

| Parameter           | Type   | Required | Description      |
| ------------------- | ------ | -------- | ---------------- |
| id                  | string | Yes      | Step identifier  |
| agent               | Agent  | Yes      | Agent instance   |
| options.description | string | No       | Step description |

**Returns:** `WorkflowStep<string, string>` - Agent workflow step

## Workflow Builder Methods

### step(stepId, step, options)

Add a step to the workflow with explicit input mapping.

| Parameter           | Type         | Required | Description                 |
| ------------------- | ------------ | -------- | --------------------------- |
| stepId              | string       | Yes      | Step identifier             |
| step                | WorkflowStep | Yes      | Step to add                 |
| options.input       | InputMapping | No       | Input mapping configuration |
| options.description | string       | No       | Step description            |

**Returns:** `Workflow<TInput, TO>` - Updated workflow builder

### then(stepId, step, options)

Add a sequential step that automatically takes input from the previous step.

| Parameter           | Type         | Required | Description                                        |
| ------------------- | ------------ | -------- | -------------------------------------------------- |
| stepId              | string       | Yes      | Step identifier                                    |
| step                | WorkflowStep | Yes      | Step to add                                        |
| options.input       | InputMapping | No       | Override input mapping (defaults to previous step) |
| options.description | string       | No       | Step description                                   |

**Returns:** `Workflow<TInput, TO>` - Updated workflow builder

### parallel(stepIds, steps, options)

Add multiple steps to execute in parallel.

| Parameter           | Type           | Required | Description                                |
| ------------------- | -------------- | -------- | ------------------------------------------ |
| stepIds             | string[]       | Yes      | Array of step identifiers                  |
| steps               | WorkflowStep[] | Yes      | Array of steps (must match stepIds length) |
| options.description | string         | No       | Parallel group description                 |

**Returns:** `Workflow<TInput, TO[]>` - Updated workflow builder with array output

### branch(condition, branches, options)

Add conditional branching to the workflow.

| Parameter           | Type                                 | Required | Description               |
| ------------------- | ------------------------------------ | -------- | ------------------------- |
| condition           | (ctx: WorkflowContext) =&gt; boolean | Yes      | Branch condition function |
| branches.true.id    | string                               | Yes      | True branch identifier    |
| branches.true.step  | WorkflowStep                         | Yes      | True branch step          |
| branches.false.id   | string                               | Yes      | False branch identifier   |
| branches.false.step | WorkflowStep                         | Yes      | False branch step         |
| options.description | string                               | No       | Branch description        |

**Returns:** `Workflow<TInput, TO>` - Updated workflow builder

### commit()

Finalize the workflow and make it executable.

**Returns:** `CommittedWorkflow<TInput, TOutput>` - Executable workflow

## CommittedWorkflow Methods

### run(input)

Execute the workflow with given input.

| Parameter | Type   | Required | Description    |
| --------- | ------ | -------- | -------------- |
| input     | TInput | Yes      | Workflow input |

**Returns:** `Promise<TOutput>` - Workflow output

## Examples

### Example 1: Basic Sequential Workflow

```typescript
import { createWorkflow, createStep } from 'primo-agent';

const doubleStep = createStep('double', async (input: number) => input * 2);
const addTenStep = createStep('addTen', async (input: number) => input + 10);

const workflow = createWorkflow({ id: 'math-pipeline' })
  .step('double', doubleStep)
  .then('addTen', addTenStep)
  .commit();

const result = await workflow.run(5);
console.log(result); // Output: 20
```

### Example 2: Using WorkflowContext

```typescript
import { createWorkflow, createStep } from 'primo-agent';

const step1 = createStep('step1', async (input: number, ctx) => {
  ctx.setState({ multiplier: 10 });
  return input * 2;
});

const step2 = createStep('step2', async (input: number, ctx) => {
  const state = ctx.getState();
  const step1Result = ctx.getResult('step1');
  return input * (state.multiplier as number);
});

const workflow = createWorkflow({ id: 'context-demo' })
  .step('step1', step1)
  .then('step2', step2)
  .commit();

const result = await workflow.run(5);
console.log(result); // Output: 100
```

### Example 3: Parallel Execution

```typescript
import { createWorkflow, createStep } from 'primo-agent';

const taskA = createStep('taskA', async (x: number) => x * 2);
const taskB = createStep('taskB', async (x: number) => x + 10);
const taskC = createStep('taskC', async (x: number) => x - 5);

const workflow = createWorkflow({ id: 'parallel-demo' })
  .parallel(['taskA', 'taskB', 'taskC'], [taskA, taskB, taskC])
  .commit();

const results = await workflow.run(20);
console.log(results); // Output: [40, 30, 15]
```

### Example 4: Conditional Branching

```typescript
import { createWorkflow, createStep } from 'primo-agent';

const checkStep = createStep('check', async (input: number) => input);
const largeStep = createStep('large', async (input: number) => input * 100);
const smallStep = createStep('small', async (input: number) => input * 2);

const workflow = createWorkflow({ id: 'branch-demo' })
  .step('check', checkStep)
  .branch((ctx) => (ctx.getResult('check') as number) > 10, {
    true: { id: 'large', step: largeStep },
    false: { id: 'small', step: smallStep },
  })
  .commit();

console.log(await workflow.run(5)); // Output: 10
console.log(await workflow.run(15)); // Output: 1500
```

### Example 5: Agent Steps

```typescript
import { createWorkflow, createAgentStep, Agent, AIAdapter, InMemoryHistory } from 'primo-agent';

const analyzer = new Agent(new AIAdapter({ model: 'gpt-4' }), new InMemoryHistory());

const summarizer = new Agent(new AIAdapter({ model: 'gpt-4' }), new InMemoryHistory());

const analyzeStep = createAgentStep('analyze', analyzer);
const summarizeStep = createAgentStep('summarize', summarizer);

const workflow = createWorkflow({ id: 'agent-pipeline' })
  .step('analyze', analyzeStep)
  .then('summarize', summarizeStep)
  .commit();

const result = await workflow.run('Analyze this article...');
console.log(result);
```

### Example 6: Message Hub for Agent Collaboration

```typescript
import { MsgHub, Agent, AIAdapter, InMemoryHistory } from 'primo-agent';

const agent1 = new Agent(new AIAdapter({ model: 'gpt-4' }), new InMemoryHistory());
const agent2 = new Agent(new AIAdapter({ model: 'gpt-4' }), new InMemoryHistory());

using msgHub = new MsgHub({
  participants: [agent1, agent2],
  announcement: { role: 'system', content: 'You are collaborators' },
});

// Listen to messages
msgHub.messages$.subscribe((msg) => {
  console.log('Received:', msg);
});

// Broadcast a message
msgHub.broadcast({ role: 'user', content: "Let's work together!" });
```

### Example 7: Pipelines

```typescript
import {
  sequentialPipeline,
  parallelPipeline,
  Agent,
  AIAdapter,
  InMemoryHistory,
} from 'primo-agent';

const agents = [
  new Agent(new AIAdapter({ model: 'gpt-4' }), new InMemoryHistory()),
  new Agent(new AIAdapter({ model: 'gpt-4' }), new InMemoryHistory()),
];

// Sequential pipeline
const seqResult = await sequentialPipeline(agents, {
  role: 'user',
  content: 'Start the task',
});

// Parallel pipeline
const parResult = await parallelPipeline(agents, {
  role: 'user',
  content: 'Each of you analyze this',
});
```
